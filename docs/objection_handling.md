# Objection Handling

| Objection | Answer | Required next step for critic |
|---|---|---|
| “This is only definition-dependent.” | Mathematical theorems are definition- and axiom-dependent. This is not a refutation. | Reject the domain axioms or provide a valid LLM counterexample. |
| “The executable layer is not empirical.” | Correct; it is a finite proof by exhaustive enumeration, not a model survey. | Show an execution failure or challenge the obligation graph formally. |
| “The six roles are arbitrary.” | The roles correspond to distinct responsibility boundaries. | Provide a superior minimal decomposition that preserves all boundaries, or a valid counterexample. |
| “Future architectures may differ.” | Architecture can differ; Layer-0 roles cannot disappear if the system remains an LLM. | Show an LLM lacking a role and every functional equivalent. |
| “Large has no fixed threshold.” | The theorem does not rely on a universal parameter cutoff. Large is a contemporary scale condition. | Show that scale ambiguity removes one Layer-0 role. |
| “RAG can answer text.” | RAG is a composite system; retriever/tool layers are not the LLM core. | Show the generative LLM layer lacks Layer 0. |
| “Encoder-only models process language.” | Language processing is broader than LLM status. | Show that an encoder-only system is ordinarily classified as an LLM and lacks Layer 0. |
